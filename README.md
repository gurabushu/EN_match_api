## ４、開発の背景
元々マッチング形式のビズネスモデルの制作物を作ろうと考えておりました。
近年、表立って活躍するサービスが双方のマッチング形式を採用しているものが多いと感じたためです。
エンジニアは勉強熱心な人が多く、またエンジニア同士の横のつながりによってモチベーションの維持や
自身のキャリアの展望、スキルの共有会など生まれるものが多いと感じたため、エンジニア向けの
マッチングアプリに路線を定めました。
また、今後のスキル向上を見据えAI相性マッチング機能を実装しました。

## 開発において工夫した点


### アプリ全般において

・UI


ログイン後とログアウト後のヘッダーの表示の切り替えと、ユーザー名の表示を実装することで
ユーザーにとって、サイトの状況が直感的にわかるようにしました。
また、テーマカラーを統一し、サイト全体のデザイン性にある種の個性を持たせたこともサイトの操作性において
より使いやすいデザインにすることができたのではないかと考えます。


・マッチング機能


githubの登録を促すことで、よりパーソナルな着眼点よりもエンジニアとしてのスキルに重きを置いた設計にしました。


・ユーザー検索機能


絞り込み機能を実装することで、各ユーザーが求める繋がりたいエンジニア像をピンポイントに表示することで
ユーザーの目的に早期にアクセスできる仕様になるよう調整しました。



### AI実装において

まず、ベーススコアを60点にし、そこから
自己紹介文の長さ、githubの登録の有無、年齢差を比較し、相性を測る機能を実装しました。
上記のそれぞれの項目の差異ごとに点数を定義し、上限95点までとすることでユーザーがより説得力と安心感を得られる機能に実装しました。


→苦労した点：実装初期、エラーや表示などがうまくいかず苦心しました。APIがたとえ落ちてもアプリが機能するように
実装したり、成功、警告、エラーをログごとに分けて記録するなど、やり方や具体的なコードを調べるのに相当な時間と労力を要しました。
まだまだ、AI実装に関しては自身でスキルとしてカウントできるレベルではないと考えておりますが、トライアンドエラーを繰り返し、
最後に実装した機能が動いた成功体験は大きな財産だと考えております。
